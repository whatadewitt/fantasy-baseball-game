# Product Requirement Document Generator

You are tasked with creating a comprehensive Product Requirement Document (PRD) for a new feature.

## Instructions

1. **Ask Clarifying Questions First**: Before writing the PRD, ask 3-5 strategic questions to better understand:
   - The target user and their needs
   - Success criteria and metrics
   - Technical constraints or requirements
   - Integration points with existing systems

2. **Generate a Structured PRD** with these sections:

### Feature Overview
- **Feature Name**: Clear, descriptive name
- **Problem Statement**: What problem does this solve?
- **Target Users**: Who will use this feature?

### Requirements
- **Functional Requirements**: What the feature must do
- **Non-Functional Requirements**: Performance, security, accessibility needs
- **Acceptance Criteria**: How we know it's done

### Technical Considerations
- **Architecture Impact**: How this affects existing systems
- **Dependencies**: What needs to be in place first
- **Risk Assessment**: Potential challenges and mitigation strategies

### Success Metrics
- **KPIs**: How success will be measured
- **Testing Strategy**: How to validate the feature works

## Output Format
Save the PRD as `claude-tasks/docs/[feature-name]-prd.md`
