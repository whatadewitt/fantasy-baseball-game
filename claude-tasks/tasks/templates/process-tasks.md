# Task Processor

You will work through the generated task list systematically.

## Current Task Context
Reference: `claude-tasks/tasks/[feature-name]-tasks.md`

## Process Rules

1. **Focus on ONE task at a time**
2. **Show your work** - Explain your approach before implementing
3. **Request review** after each task completion
4. **Mark completed tasks** with ✅ in the task list
5. **Update progress** by modifying the task file

## Implementation Steps

For each task:
1. Read the current task requirements
2. Explain your implementation approach
3. Show the code changes you'll make
4. Implement the changes
5. Update the task list to mark completion
6. Ask for review before proceeding

## Status Updates

After completing each task, update the task file:
- Change `[ ]` to `✅` for completed tasks
- Add completion notes if needed
- Identify any issues or dependencies discovered

## Review Protocol

After each task, ask:
"Task [X] completed. Please review the changes. Reply 'continue' to proceed to the next task, or provide feedback for adjustments."
